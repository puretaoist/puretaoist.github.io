五、知识总结

组件 作用 本实验中的体现
Node.js JavaScript 运行时环境 执行 Hexo 框架和 npm 命令
npm Node.js 包管理器 安装 Hexo CLI 和项目依赖
Git 版本控制系统 克隆主题模板、配置用户信息
Hexo CLI 博客框架命令行工具 初始化项目、创建文章、生成静态页面、本地预览
Markdown 轻量级标记语言 编写博客文章内容

本实验使用的 Hexo 是静态博客生成器，它将 Markdown 源文件预编译为纯 HTML 页面，具有加载速度快、安全性高、无需数据库的特点。

六、出现问题

问题现象 可能原因 解决方法
‘hexo’ 不是内部或外部命令 Hexo CLI 未正确安装 执行 npm install -g hexo-cli 重新安装，装完后重启 CMD
hexo init 失败/速度极慢 网络无法正常访问 GitHub 使用 Gitee 镜像源
npm install 速度极慢 默认使用国外镜像源 切换为国内镜像源

七、心得体会

通过本次实验，首次接触并搭建了完整的 Web 开发环境。从 Node.js 的安装到 Hexo 博客项目的初始化，我深刻理解了 Web 开发中”环境准备”这一环节的重要性。特别是在解决 hexo init 网络问题时，我认识到国内开发者需要灵活运用镜像源来应对网络环境的问题。此外，Hexo 作为一款优秀的开源静态博客框架，让我体会到了开源社区的强大和便捷——无需购买服务器和域名，就能在本地拥有一个完整的博客系统。
